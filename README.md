# SunoCraft — Creator Studio 🎬🎵

SunoCraft is a fully client-side, browser-based audio and video editor designed to easily stitch audio files (such as tracks generated by Suno AI) together, trim them, and compile them into a video with a looped background image or GIF.

Because it uses **FFmpeg.wasm** (WebAssembly), all processing happens directly in your browser's memory. **No files are uploaded to any server.**

---

## ✨ Features

- **100% Local & Secure:** All file stitching, trimming, and video encoding happen directly inside your browser. Your media never leaves your computer.
- **Drag & Drop Timeline:** Easily reorder your audio tracks using a smooth drag-and-drop interface powered by `@dnd-kit/react`.
- **Precise Trimming:** Set custom start and end times for each audio track with an interactive visual range helper.
- **Multiple Output Formats:**
  - **🎬 MP4 Video:** Stitches your audio tracks and loops a background image/GIF for the exact duration of the audio.
  - **🎵 MP3/WAV Audio:** Concatenates all audio tracks into a single MP3 or high-fidelity WAV file.
- **Modern UI:** Sleek dark-themed design with premium glassmorphism accents, hover micro-animations, and a gradient progress bar.

---

## 🚀 Tech Stack

- **Framework:** React 18 + TypeScript + Vite
- **Styling:** Tailwind CSS v3
- **Processing Engine:** `@ffmpeg/ffmpeg` (FFmpeg.wasm v0.12.x)
- **Drag & Drop:** `@dnd-kit/react` + `@dnd-kit/dom`

## 🛠️ Getting Started

### Prerequisites
Make sure you have [Node.js](https://nodejs.org/) installed (v18 or higher recommended).

### Installation

1. Clone the repository:
   ```bash
   git clone https://github.com/hikmetbozkurt/SunoCraft.git
   cd SunoCraft
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Run the development server:
   ```bash
   npm run dev
   ```

4. Open your browser and navigate to: `http://localhost:5173/`

## 🔒 Cross-Origin Isolation Note

FFmpeg.wasm relies on `SharedArrayBuffer` to run multi-threaded processes, which requires specific HTTP headers sent by the hosting server for security (Cross-Origin Isolation):

- `Cross-Origin-Opener-Policy: same-origin`
- `Cross-Origin-Embedder-Policy: require-corp`

The development server (`vite.config.ts`) is pre-configured to send these headers automatically. If you deploy this project to production (e.g., GitHub Pages, Vercel), make sure your hosting platform serves these headers or use a service worker helper (like `coi-serviceworker`).
